## üìã Executive Summary

**Tanggal Audit**: 9 Oktober 2025
**Status**: ‚úÖ **DITEMUKAN RACE CONDITION KRITIS**
**Severity**: üî¥ **HIGH** - Mempengaruhi user experience dan consistency data

### üéØ Temuan Utama

1. **‚úÖ Token tidak segera berkurang** setelah submit meski response sukses mengembalikan `remainingTokens: 3`
2. **‚ùå Error `INVALID_TOKEN`** muncul tak lama setelah submit
3. **‚úÖ Assessment tetap berhasil diproses** setelah kembali ke dashboard (token akhirnya berkurang)
4. **üêõ Root Cause**: Frontend tidak men-trigger token balance update hingga user action berikutnya

### **Masalah #2: Cache Invalidation vs Refetch**

```typescript
// FILE: src/utils/cache-invalidation.ts
export async function invalidateTokenBalanceCache(userId: string): Promise<void> {
  // 1. ‚úÖ Clear localStorage cache
  localStorage.removeItem(`tokenBalanceCache_${userId}`);
  
  // 2. ‚úÖ Clear SWR cache with revalidate
  await mutate('/api/auth/token-balance', undefined, { revalidate: true });
  
  // 3. ‚úÖ Clear IndexedDB cache
  await indexedDBCache.delete(`token-balance-${userId}`);
  
  // ‚ùå MASALAH: Jika tidak ada component yang actively watching
  // endpoint `/api/auth/token-balance`, maka revalidate tidak trigger fetch baru!
}
```

**Akar Masalah**: 
- SWR `mutate()` dengan `revalidate: true` hanya refetch jika ada **active hook** yang subscribe ke key tersebut
- Di halaman `/assessment-loading`, tidak ada hook yang listen ke `/api/auth/token-balance`
- Token balance baru hanya akan diambil saat user navigate ke dashboard (yang punya `useUserData` hook)

## üéØ Identified Race Conditions

### **Race Condition #1: Submit ‚Üí Navigate ‚Üí Token Update**

```
Timeline:
‚îú‚îÄ T+0ms:    Submit button clicked
‚îú‚îÄ T+20ms:   Navigation to /assessment-loading
‚îú‚îÄ T+100ms:  submitFromAnswers() called
‚îú‚îÄ T+1000ms: Backend response + token deducted
‚îú‚îÄ T+1010ms: onTokenBalanceUpdate() called
‚îî‚îÄ T+1015ms: Cache invalidated (but page already loaded with old cache)

ISSUE: Navigation happens BEFORE token is updated in DB
RESULT: Loading page shows old token balance
```

**Impact**: Medium - User sees stale token count temporarily

---

### **Race Condition #2: Cache Invalidation ‚Üí No Active Listener**

```
Timeline:
‚îú‚îÄ T+1010ms: onTokenBalanceUpdate() called
‚îú‚îÄ T+1011ms: invalidateTokenBalanceCache(userId)
‚îú‚îÄ T+1012ms: mutate('/api/auth/token-balance', undefined, { revalidate: true })
‚îú‚îÄ T+1013ms: ‚ùå No active SWR hook listening to this key!
‚îî‚îÄ T+1014ms: Cache cleared but no refetch triggered

ISSUE: SWR revalidate only works if there's an active hook
RESULT: Token balance not refreshed until next page load
```

**Impact**: High - Token balance stays stale until user navigates

---

### **Race Condition #3: Monitoring Fetch ‚Üí Old Token**

```
Timeline:
‚îú‚îÄ T+1020ms: WebSocket/polling monitoring started
‚îú‚îÄ T+2000ms: Intermediate status fetch
‚îú‚îÄ T+2001ms: Uses token from localStorage (might be old session token?)
‚îú‚îÄ T+2100ms: Backend: "INVALID_TOKEN" error
‚îî‚îÄ T+5000ms: Assessment completes anyway (backend job continues)

ISSUE: Monitoring might use stale/invalid token for status checks
RESULT: Error appears but doesn't affect assessment processing
```

**Impact**: Low - Cosmetic error, assessment still completes
